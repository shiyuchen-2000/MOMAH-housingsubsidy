MOMAH（住房部）/ 住房支援署 —— *Dynamic Subsidy Allocation & Optimization* 的可交互前端演示。

## 一、如何打开

**方式 ①：直接演示（无需安装，推荐用于现场）**
双击本目录下的 **`standalone.html`**。它通过 CDN 加载 React / Recharts，双击即在浏览器运行（需联网以加载 CDN 与字体；离线时图表会降级为占位，其余功能与数字照常）。

**方式 ②：工程化（Vite 项目）**
```bash
npm install
npm run dev        # 本地开发，按提示打开 http://localhost:5173
npm run build      # 生产构建，产物在 dist/
npm run preview    # 预览构建产物
```

## 六、数据模型与严谨性

所有数字锚定 BRD V0.5.1，并相互勾稽。引擎与数据已用 Node 单元测试验证（见 `tests/engine.test.js`）。

**关键锚点**：合格家庭 140 万；历史基线 127,952 合同 / 180.98 亿 SAR / 件均 141,444（2024–2025/07）；第三阶段预算 79 亿 / 5 年；目标 510,000 合同（REDF 31 万 / ZATCA 15 万 / 发展性 5 万）；HBR 基线 40–41%、目标 30–35%；自有率 66.24%→70%。

**双框架（重要）**：BRD 自身存在两套口径——历史基线 18B/12.8 万件（件均 14.1 万，含一次性/贷款类总支援），与第三阶段“弹性预算” 7.9B/51 万件（件均约 1.5 万）。本 DEMO 据此拆为两个**互不混用**的子模型：
- **配分 / 节省 / 公平性** 在“弹性预算”框架（`subsidyBase`，年支出约 1.9B，节省以 79 亿预算为分母，对应 BRD 的 17–43%）。
- **HBR** 在“套餐/买断支援”框架（`pkgBase` ≈ 9.5–17 万，按揭本金买断），因为只有大额买断才能实质降低月供与负担。

**公式**：
- Fairness Gap `FG = (10K以下补贴占比) ÷ (10K以下人口占比)`，`FG ≥ 1.0` 视为公平。
- HBR = 月供 ÷ 月净收入；月供按 `(房价×(1-首付) - 套餐买断)` 以 5.2% 年化、25 年、10% 首付计算（已校准至基线 ≈40.5%）。
- 节省 = 当前矩阵年支出 − 情景年支出（×5 为五年）。

**经验证的可达区间**（滑块组合）：节省 1.2–3.55B（最高约 45% 预算）；Fairness Gap 0.58 → ≥1.0；HBR 40.5% → ~33.9%。三者存在真实权衡（提升公平/降负担需投入，会减少节省），正是 What-if 的演示价值。

## 七、源码结构

```
standalone.html      # 双击即开的成品（含完整 React 源码，内联 + CDN）
index.html           # Vite 项目入口（npm run dev/build）
app.html             # 打开方式说明页（静态）
package.json         # 依赖与脚本
vite.config.js       # Vite 配置（React 插件，输出 dist/）
src/
  main.jsx           # 渲染入口
  App.jsx            # 全部组件（数据层 / 引擎 / i18n / 三角色页面）
  styles.css         # DGA 风格样式（含 RTL）
scripts/
  build-standalone.mjs # 由 src/ 生成 standalone.html
tests/
  engine.test.js     # 引擎/数据 Node 单元测试
```
**唯一源是 `src/`**。`standalone.html` 由 `src/App.jsx` + `src/styles.css` 生成：修改逻辑后运行 `node scripts/build-standalone.mjs` 重新生成即可，二者自动保持同步。
